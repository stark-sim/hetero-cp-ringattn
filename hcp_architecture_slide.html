🔷 异构场景下合作：P2P Ring 网络模型

  • 传统 CP 采用 Collective (all-gather/all-to-all)，依赖 NCCL 和同构 GPU
  • HCP 回归 Ring Attention 论文原始 P2P 设计：每个 domain 只与邻居通信
  • 发送 K/V block 到 next host，同时从 previous host 接收 K/V block
  • online softmax 增量更新，支持任意互联拓扑（TCP/QUIC/RDMA）

🔶 非均等显存：Capacity-Aware 分片

  • Worker handshake 上报 device 可用显存，Coordinator 按 capacity 比例分配
  • 1:1 均分 → Pearl 16GB OOM（layer 17/24 失败）❌
  • 3:1 分片 → White 750K / Pearl 250K → 成功 ✅
  • 这是异构长 context 的必需策略

🔹 Ring-attn P2P 网络模型

  • Micro KV Block 细粒度切分：防止大 block 传输死锁
  • Pipeline Overlap：compute 与 network I/O 重叠
  • QUIC Transport：单 connection + per-layer bidirectional stream
  • 并发 send+recv (tokio::join!) 避免 ring 死锁
